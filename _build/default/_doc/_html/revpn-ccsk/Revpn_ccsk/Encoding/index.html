Module Revpn_ccsk.Encoding

Encoding from CCS nets into CCS processes.

This module implements Algorithm 1 of:

Hernán C. Melgratti, Claudio Antares Mezzina, G. Michele Pinna. "Encoding Reversible Petri nets into CCSK."

Given a CCS net (a Net.labelled_net in which every transition has at most two places in its preset, and exactly two only when its label is tau) together with a marking, the algorithm produces:

  • a partial mapping of defining equations D, associating to every place s of the net a process variable Xs whose defining equation describes the possible evolutions of s (Step 1-3 of the algorithm); synchronising transitions additionally introduce a fresh restricted action (Step 5-8);
  • a CCS process Q, namely the parallel composition of (as many copies as the marking prescribes of) the place variables, restricted on the fresh synchronisation actions (Step 10-11).

Theorem 1 of the paper states that the marking graph of the CCS net system is bisimilar to the LTS generated by Q under D.

val var_of_place : Net.place_id -> string

The process variable Xs associated to a place s.

val var_of_trans : Net.transition_id -> string

The auxiliary variable Yt associated to a transition t, used while the equation of a place is being built (Step 2) before being substituted away (Steps 3 and 7-8).

val action_of_sync : Net.transition_id -> string

The fresh restricted action st associated to a synchronising transition t (Step 5-8 of Algorithm 1).

val label_of_transition : Net.labelled_net -> Net.transition_id -> string

Look up the action lambda(t) labelling a transition, as recorded in net.label. We follow the convention used throughout the test-suite: the label "tau" (or the empty label) denotes the silent action, any other string a denotes the input action a, and a label of the form "!a" denotes the output (co-)action a.

val act_of_label : string -> Ccsk.act
val is_sync : Net.labelled_net -> Net.transition_id -> bool

is_sync net t holds when |•t| = 2, i.e. t is a synchronising transition in the sense of Net.Definition 7 (CCS nets).

val choice_of_list : Ccsk.CCS.process list -> Ccsk.CCS.process

choice_of_list ps builds the (possibly empty) choice Y1 + .. + Yk out of a non-empty list of processes; the empty list yields Zero, matching the case of a place with no outgoing transitions.

val parallel_of_list : Ccsk.CCS.process list -> Ccsk.CCS.process

parallel_of_list ps builds X1 | .. | Xn out of a (possibly empty) list of processes; the empty list yields Zero (the postset of a transition is empty exactly when it produces no tokens, which does not occur for transitions of a CCS net but is handled defensively).

val init_place_equations : Net.labelled_net -> Ccsk.CCS.equations

Step 2: initialise D(Xs) = Yt1 + .. + Ytk for every place s, where t1,..,tk are the transitions in the postset of s.

val subst_in_equations : string -> Ccsk.CCS.process -> Ccsk.CCS.equations -> Ccsk.CCS.equations

Substitute every occurrence of variable x by repl in the right-hand side of every equation of d (used to eliminate the auxiliary Yt variables once their definition has been computed).

val encode_simple_transitions : Net.labelled_net -> Ccsk.CCS.equations -> Ccsk.CCS.equations

Step 3: for every non-synchronising transition t (single place s_star in its preset), substitute Yt in D(Xs_star) by lambda(t).(Xs1 | .. | Xsn) where s1,..,sn is the postset of t.

val subst_var_in_one_equation : string -> Ccsk.CCS.process -> string -> Ccsk.CCS.equations -> Ccsk.CCS.equations

subst_var_in_one_equation x repl x_eq d substitutes x by repl only in the right-hand side of the equation for variable x_eq, leaving every other equation untouched. Used by encode_sync_transitions, where the same auxiliary variable Yt must be resolved differently depending on which of the two equations (preset places) mentions it: the place that "opens" the synchronisation gets the continuation, the place that "closes" it gets at.0.

val encode_sync_transitions : Net.labelled_net -> Ccsk.CCS.equations -> Ccsk.CCS.equations * string list

Step 5-8: for every synchronising transition t with preset {s1, s2}, pick a fresh action at. The equation D(Xs1) (which mentions Yt as one of its summands) is updated by substituting Yt with at.(Xs1' | .. | Xsn') -- the continuation, where s1',..,sn' is the postset of t -- while D(Xs2) is updated by substituting Yt with at.0. Returns the updated equations together with the set of fresh actions introduced.

val replicate : int -> Ccsk.CCS.process -> Ccsk.CCS.process

replicate n p builds the parallel composition of n copies of p (X^n in the notation of the paper); replicate 0 p = Zero.

val assemble_marking : Net.labelled_net -> Net.marked_net -> Ccsk.CCS.process

Step 11: Q = X_{s1}^{m(s1)} | .. | X_{sn}^{m(sn)}, skipping unmarked places (whose contribution is Zero and would otherwise clutter the process with inert summands).

val restrict_all : string list -> Ccsk.CCS.process -> Ccsk.CCS.process

Step 11: restrict Q on every fresh synchronisation action.

type result = {
  1. process : Ccsk.CCS.process;
  2. equations : Ccsk.CCS.equations;
}
val encode : Net.marked_net -> result

encode (net, m) runs Algorithm 1 on the CCS net net with marking m, producing the CCS process Q and the (now Yt-free) defining equations D such that, by Theorem 1, the marking graph of (net, m) is bisimilar to the LTS generated by Q under D.
